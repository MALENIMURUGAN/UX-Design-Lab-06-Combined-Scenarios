# UX-Design-Lab-06-Combined-Scenarios

##  **Aim**

To apply UX design principles by performing solution ideation, creating user stories, developing user scenarios, designing task flows, and structuring information architecture for a hospital appointment booking system.


## **Algorithm / Procedure**

### Step 1: Identify Problem

* Selected UX problem: *Booking an appointment in a hospital app*


### Step 2: Perform Ideation (Crazy 8s)

* Generated 8 quick ideas:

  1. One-tap booking
  2. AI symptom checker
  3. Voice booking
  4. Calendar-based slot view
  5. Emergency fast booking
  6. Doctor recommendation system
  7. Family profile booking
  8. Chat assistant



### Step 3: Create User Stories

* Defined user needs using standard format:

```
As a working professional, I want to quickly book an appointment, so that I save time.
As a parent, I want to book for my children, so that I can manage their health easily.
As a first-time user, I want to see reviews, so that I can trust the doctor.
As a senior citizen, I want voice booking, so that I avoid typing.
As an urgent patient, I want the nearest doctor, so that I get quick treatment.
```

---

### Step 4: Develop Scenario

* Selected user: Working professional
* Created a narrative describing:

  * Context: Busy workday
  * Emotion: Stress, urgency
  * Action: Uses quick booking
  * Outcome: Fast appointment confirmation



### Step 5: Create Task Flow

```
Start
  ↓
Open App
  ↓
Login
  ↓
Book Appointment
  ↓
Enter Symptoms
  ↓
View Doctors
  ↓
Select Doctor
  ↓
Check Availability
  ↓
[Decision: Slot Available?]
   ↓ Yes → Confirm Booking
   ↓ No  → Show Other Slots
  ↓
Payment
  ↓
End
```


### Step 6: Design Information Architecture

**Categories Identified:**

* Appointments
* Doctors
* Health Records
* Payments
* Support
